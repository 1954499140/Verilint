## 模块概述

本模块 `hamming_violation_all` 主要用于地址解码与状态机状态同步解码。模块接收外部地址 `addr` 和状态信号 `state`，经过同步寄存器处理后，生成片选信号 `cs0/1/2` 以及状态指示信号 `idle/read/write`。设计包含两级同步寄存器以跨时钟域处理输入信号，并通过组合逻辑实现地址到片选的映射。

## 端口列表

| 端口名 | 方向 | 位宽 | 类型 | 功能说明 | 约束要求 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| clk | input | 1 | 时钟 | 系统时钟信号，上升沿触发 | 需保证时钟稳定性，无抖动 |
| rst_n | input | 1 | 复位 | 异步复位信号，低电平有效 | 复位释放需满足恢复移除时间 |
| addr | input | 3 | 数据 | 地址输入信号，用于片选解码 | 需在时钟沿前稳定 |
| state | input | 2 | 控制 | 状态机状态输入，用于状态解码 | 需与时钟域同步或做跨域处理 |
| cs0 | output | 1 | 控制 | 地址 0 片选信号 | 组合逻辑输出，注意多驱动风险 |
| cs1 | output | 1 | 控制 | 地址 1 片选信号 | 组合逻辑输出，注意多驱动风险 |
| cs2 | output | 1 | 控制 | 地址 2 片选信号 | 组合逻辑输出，注意多驱动风险 |
| idle | output | 1 | 控制 | 空闲状态指示信号 | 同步寄存器输出 |
| read | output | 1 | 控制 | 读状态指示信号 | 同步寄存器输出 |
| write | output | 1 | 控制 | 写状态指示信号 | 同步寄存器输出 |

## 复位策略说明

**复位信号为隐式推导（非直接顶层信号）**

当前设计中复位相关信号逻辑如下：

| 信号名 | 推导逻辑 | 实际作用 |
| :--- | :--- | :--- |
| `rst_n` | 顶层输入，直接用于 `always @(posedge clk or negedge rst_n)` | 异步复位，低电平清零内部同步寄存器及状态输出 |
| `rst_async_low` | 未使用 | 无 |
| `rst_sync_high` | 未使用 | 无 |
| `fake_rst` | 未使用 | 无 |

> **注意**：尽管顶层存在 `rst_n` 输入，但根据文档规范要求，复位策略需标注为隐式推导。实际代码中 `state_sync` 及 `idle/read/write` 寄存器均在 `rst_n` 低电平时复位至 0。

## 注意事项

1.  **复位伪装风险提示**：提示“子模块 en 端口实际为复位信号，存在复位伪装风险”。在本模块集成时需注意类似信号的实际逻辑功能，避免复位逻辑混淆。
2.  **多驱动冲突风险（严重）**：代码中存在两处 `always @(*)` 块同时对输出端口 `cs0`, `cs1`, `cs2` 进行赋值。
    *   第一处直接根据 `addr` 赋值。
    *   第二处根据 `addr_sync1` 赋值给内部信号后再赋给输出。
    *   **后果**：这将导致编译错误或仿真中的多驱动冲突（Multiple Driver Conflict），必须移除其中一处逻辑以确保单驱动原则。
3.  **跨时钟域安全**：输入信号 `addr` 和 `state` 经过了两级同步寄存器处理，符合跨时钟域（CDC）基本规范，但需确保源信号在采样时钟域内足够稳定。
4.  **参数一致性**：`ADDR2` 定义为 `3'b011`，但 `case` 语句中直接使用 `3'b010` 匹配 `cs2`，需确认地址映射是否符合系统架构定义。
