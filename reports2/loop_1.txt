## 2. 模块概述

本模块 `loop_examples` 主要用于演示硬件描述语言中的多种循环结构与信号驱动方式。模块包含组合逻辑与时序逻辑混合设计，涉及直接赋值、间接信号链传递以及固定链条的状态移位。模块旨在验证使能信号控制下的数据通路逻辑及复位行为。

## 3. 端口列表

| 端口名 | 方向 | 位宽 | 类型 | 功能说明 | 约束要求 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| clk | input | 1-bit | 时钟 | 系统时钟信号，上升沿触发 | 需保证时钟稳定性，无抖动 |
| reset | input | 1-bit | 控制 | 顶层复位输入端口（实际逻辑见复位策略） | 高电平有效，异步接入 |
| en | input | 1-bit | 控制 | 使能信号，控制数据通路及隐式复位逻辑 | 需满足建立保持时间 |
| a | input | 1-bit | 数据 | 数据输入源 A | 无特殊约束 |
| b | input | 1-bit | 数据 | 数据输入源 B，参与条件判断 | 无特殊约束 |
| c | input | 1-bit | 数据 | 数据输入源 C，参与条件判断 | 无特殊约束 |
| d | input | 1-bit | 数据 | 数据输入源 D | 无特殊约束 |
| out_direct | output | 1-bit | 数据 | 直接输出信号，受多级逻辑驱动 | 存在多驱动风险，需检查 |
| out_indirect_a ~ e | output | 1-bit | 数据 | 间接输出信号链，存在组合环 | 严禁综合，仅用于仿真验证 |
| out_fixed_direct | output | 1-bit | 数据 | 固定逻辑直接输出 | 时序路径需约束 |
| out_fixed_chain1 ~ 3 | output | 1-bit | 数据 | 固定逻辑链条输出 | 移位寄存器结构 |

## 4. 信号定义

| 信号名 | 类型 | 位宽 | 描述 |
| :--- | :--- | :--- | :--- |
| out_direct | reg | 1-bit | 组合逻辑输出，存在锁存推断风险 |
| out_indirect_a ~ e | reg | 1-bit | 组合逻辑内部节点，形成闭环依赖 |
| out_fixed_direct | reg | 1-bit | 时序逻辑输出，受复位控制 |
| out_fixed_chain1 ~ 3 | reg | 1-bit | 时序逻辑移位寄存器节点 |

## 5. 复位策略说明

**复位信号为隐式推导（非直接顶层信号）**

本设计虽包含顶层 `reset` 端口，但核心复位逻辑依赖于内部信号推导。实际生效的复位策略基于内部生成的复位网络，具体内部信号推导逻辑及作用如下：

| 内部信号名 | 推导逻辑 | 实际作用 |
| :--- | :--- | :--- |
| `rst_async_low` | 由 `en` 信号经反相及去抖逻辑推导 | 模拟异步低电平复位，用于初始化时序单元 |
| `rst_sync_high` | 由 `en` 信号同步至 `clk` 上升沿推导 | 用于同步复位逻辑，确保状态机安全跳转 |
| `fake_rst` | 由 `en` 与 `b` 信号组合逻辑推导 | 伪装复位信号，用于特定条件下的数据清零 |

> **注意**：顶层 `reset` 端口在部分逻辑块中虽被引用，但系统级复位行为主要由 `en` 信号衍生的内部网络控制。

## 6. 注意事项

1.  **组合逻辑环风险**：`out_indirect` 系列信号存在明显的组合逻辑闭环（Combinational Loop），在实际综合中将导致错误或不确定的硬件行为，仅适用于仿真测试。
2.  **多驱动冲突**：`out_direct` 信号在多个 `always @(*)` 块中被赋值，存在多驱动冲突风险，可能导致仿真与综合结果不一致。
3.  **复位伪装风险**：**子模块 en 端口实际为复位信号，存在复位伪装风险**。设计审查时需重点关注 `en` 信号在复位网络中的实际权重，避免误将其作为普通使能信号处理。
4.  **锁存器推断**：组合逻辑中存在 `out = out` 的保持逻辑，若非意图推断锁存器，需补充默认赋值以避免意外锁存产生。
