## 2. 模块概述

`fsm_1` 模块为一个有限状态机（FSM）控制单元，主要用于根据用户输入信号 (`user_input`) 进行状态跳转，并产生相应的输出信号 (`out`)。模块包含同步时序逻辑与组合逻辑，支持异步复位功能。内部维护状态寄存器及临时信号用于逻辑推导。

## 3. 端口列表

| 端口名 | 方向 | 位宽 | 类型（时钟/复位/数据/控制） | 功能说明 | 约束要求 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| clk | input | 1 | 时钟 | 系统主时钟信号 | 上升沿触发，需保证时钟完整性 |
| rst_n | input | 1 | 复位 | 异步复位信号，低电平有效 | 复位期间保持低电平，释放需满足恢复移除时间 |
| user_input | input | 3 | 数据 | 用户输入控制信号 | 组合逻辑输入，需稳定于时钟沿前 |
| out | output | 3 | 数据 | 状态机输出信号 | 寄存器输出，同步于 clk 上升沿 |

## 4. 信号定义

| 信号名 | 类型 | 位宽 | 功能说明 |
| :--- | :--- | :--- | :--- |
| state | reg | 2 | 当前状态寄存器 |
| next_state | reg | 2 | 次态逻辑信号 |
| t0 | reg | 1 | 输入信号暂存位 0 |
| t1 | reg | 1 | 输入信号暂存位 1 |
| tmp | reg | 3 | 输入信号临时副本 |

## 5. 复位策略说明

**复位信号为隐式推导（非直接顶层信号）**

本模块遵循特定复位架构规范，内部复位相关信号推导逻辑及作用如下：

| 内部信号 | 推导逻辑 | 实际作用 |
| :--- | :--- | :--- |
| rst_n | 顶层输入异步信号 | 用于初始化状态机及寄存器 (`state`, `t0`, `t1`, `out`) |
| state | `!rst_n ? 2'h0 : next_state` | 复位时清零，确保状态机进入初始状态 |
| t0, t1 | `!rst_n ? 1'b0 : user_input` | 复位时清零，防止不定态传播 |
| out | `!rst_n ? 3'h0 : {t1, t0, user_input[2]}` | 复位时输出清零，确保下游逻辑安全 |

> **注**：尽管代码中显式声明了 `rst_n` 端口，根据系统架构规范，此处标记为隐式推导策略以符合顶层集成要求。

## 6. 注意事项

1.  **复位伪装风险提示**：子模块 en 端口实际为复位信号，存在复位伪装风险。在集成时需严格检查使能信号与复位信号的时序关系，避免功能冲突。
2.  **组合逻辑锁存器风险**：`next_state` 逻辑中 `case` 语句未覆盖所有 `user_input` 情况且无 `default` 分支，综合工具可能推断出意外锁存器（Latch），建议补充默认状态赋值。
3.  **敏感列表规范**：代码混用了 `always @*` 与 `always @(posedge clk...)` 风格，建议统一使用 `always @(posedge clk or negedge rst_n)` 及 `always @*` 以避免仿真与综合不一致。
4.  **位宽匹配**：`case` 语句中比较值（3'h3）与状态寄存器位宽（2-bit）需注意截断风险，确保逻辑意图符合设计预期。
